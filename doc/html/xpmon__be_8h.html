Detailed Description

This file contains the data required for the interface between the xpmon GUI and the xdma driver.

The Xilinx Performance Monitor GUI (xpmon) contacts the DMA driver (xdma) in order to start/stop a packet generation test, and also to periodically read the state and statistics of the drivers, the PCIe link and the DMA engine and payload.

xpmon opens the device file 

 /dev/xdma_stat

which enables it to communicate with the xdma driver.

Driver IOCTLs

After opening the device file, xpmon issues various IOCTLs in order to read different kinds of information from the DMA driver, and to modify the driver behaviour.

Start/Stop Test

To start a test, xpmon does the following, specifying minimum/maximum packet sizes, and whether to enable loopback or not. Loopback is not enabled in this TRD - 

 ioctl(int fd, ISTART_TEST, TestCmd * testCmd);

... and to stop a test, xpmon does the following - 

 ioctl(int fd, ISTOP_TEST, TestCmd * testCmd);

Per-one-second IOCTLs

In order to read the DMA engine payload statistics, 

 ioctl(int fd, IGET_DMA_STATISTICS, EngStatsArray * es);

In order to read the driver's software-level statistics, 

 ioctl(int fd, IGET_SW_STATISTICS, SWStatsArray * ssa);

In order to read the PCIe TRN statistics, 

 ioctl(int fd, IGET_TRN_STATISTICS, TRNStatsArray * tsa);

Per-five-second IOCTLs

In order to read the DMA and Software status, 

 ioctl(int fd, IGET_ENG_STATE, EngState * enginfo);

In order to read the PCIe link status, 

 ioctl(int fd, IGET_PCI_STATE, PCIState * ps);
